## 2026-03-07 飞书 API 缓存优化（修复版）

**问题：** OpenClaw Gateway 每分钟健康检查调用飞书 `/open-apis/bot/v3/info` API，导致每月约 4.3 万次调用，超出飞书企业自建应用 1万/月的额度限制。

**来源：** 用户分享文章 https://xx0a.com/blog/openclaw-feishu

**第一次修复（未生效）：**
- 修改了 `/root/.openclaw/extensions/feishu/src/probe.ts`
- 但 Gateway 实际使用的是系统自带的 `/usr/lib/node_modules/openclaw/extensions/feishu/src/probe.ts`

**第二次修复（正确版本）：**
- **涉及文件：** `/usr/lib/node_modules/openclaw/extensions/feishu/src/probe.ts`
- **备份信息：** `probe.ts.backup.20260307_132006`
- **修改内容：** 添加 24 小时缓存机制（PROBE_CACHE_TTL_MS = 24 * 60 * 60 * 1000），缓存键基于 appId 和 domain
- **操作记录：** 
  1. 备份系统自带版本
  2. 应用缓存修改
  3. 重启 Gateway（SIGUSR1 平滑重启）

**预期效果：** API 调用从 ~1440 次/天降至 ~1 次/天

**后续跟进：** 建议 1-2 小时后检查飞书后台 API 调用统计
