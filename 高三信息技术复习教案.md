# 高三信息技术基础综合复习教案

## 一、基本信息

| 项目 | 内容 |
|------|------|
| **课题** | 信息技术应用基础综合复习 |
| **课型** | 高三复习课 |
| **课时** | 3课时（2节课堂教学 + 1节上机实践） |
| **授课对象** | 高三年级学生 |
| **教学内容来源** | 整合《走进信息时代》《认识信息系统》《选配信息技术设备》《使用操作系统》四单元 |

---

## 二、教学目标

### 知识与技能
1. 系统梳理信息技术发展历程、信息社会特征及相关法律法规
2. 掌握信息系统组成结构、数制转换方法及信息编码原理
3. 理解计算机硬件性能指标含义，能根据需求合理选配设备
4. 熟练操作系统安装配置、应用程序管理及常用文字输入方法

### 过程与方法
1. 通过知识图谱构建，建立信息技术基础知识的整体框架
2. 通过典型案例分析，提升信息鉴别与问题解决能力
3. 通过真题演练，掌握高考信息技术基础模块的答题技巧

### 情感态度与价值观
1. 强化信息社会责任意识，自觉践行网络文明规范
2. 增强对国产信息技术发展的认同感，树立科技报国志向

## 四、课时安排

### 第1课时：信息社会与信息系统（课堂教学）

#### 【复习目标】
- 梳理信息技术发展历程及信息社会特征
- 掌握信息系统组成及工作原理
- 理解信息编码与数制转换

#### 【教学过程】

**环节一：知识框架构建（10分钟）**

教师活动：
- 展示"信息技术基础"知识思维导图
- 引导学生回忆四个单元的核心概念关联

学生活动：
- 补充完善个人复习笔记中的知识框架
- 提出存疑知识点

**环节二：重点知识精讲（20分钟）**

| 知识点 | 讲解要点 | 典型例题 |
|--------|----------|----------|
| **信息技术发展阶段** | 语言→文字→印刷→电报电话→广播电视→计算机互联网→新一代信息技术 | 判断某技术属于哪个发展阶段 |
| **信息社会特征** | 网络化、数字化、智能化；信息经济、信息文化、信息治理 | 信息社会行为的法律与道德辨析 |
| **信息系统组成** | 硬件系统（输入/处理/输出/存储）、软件系统（系统/应用）、数据、用户 | 分析某应用场景中的系统组成要素 |
| **数制转换** | 十进制↔二进制（除2取余/乘2取整）、二进制↔十六进制（4位一组） | 计算题：将(10010110)₂转换为十六进制 |
| **信息编码** | ASCII（128字符/7位）、Unicode、图像分辨率与存储容量计算 | 图像文件大小计算：800×600像素24位色图像的存储容量 |

**环节三：真题演练（10分钟）**

例题1（信息社会）：
> 下列行为中，符合信息社会道德规范的是（ ）
> A. 在社交平台随意转发未经核实的信息
> B. 下载试用版软件后破解长期使用
> C. 引用网络资源时注明出处和作者
> D. 擅自修改他人电脑中的文件

例题2（数制转换）：
> 十进制数100转换为二进制是（ ）
> A. 01100100  B. 01100101  C. 1100100  D. 1100110

例题3（存储计算）：
> 一张分辨率为1024×768的24位真彩色图像，未经压缩时的存储容量约为（ ）
> A. 2.25MB  B. 2.25KB  C. 7.5MB  D. 0.75MB

**环节四：易错点归纳（5分钟）**

- ⚠️ 混淆：信息技术"发展特征"vs"信息社会特征"
- ⚠️ 易错：二进制与十六进制转换时位数分组（整数部分从右往左，小数部分从左往右）
- ⚠️ 注意：存储容量计算中单位换算（1KB=1024B，1MB=1024KB）

**环节五：课后任务**
- 完成数制转换专项练习10题
- 整理信息系统组成结构图

---

### 第2课时：硬件设备与操作系统（课堂教学+演示）

#### 【复习目标】
- 掌握核心硬件设备功能与性能指标
- 理解操作系统功能及国产系统发展
- 学会设备选配与性价比分析

#### 【教学过程】

**环节一：硬件知识梳理（15分钟）**

| 硬件类别 | 核心性能指标 | 选配要点 |
|----------|--------------|----------|
| **CPU** | 主频(GHz)、核心数、缓存 | 办公vs游戏vs专业设计的不同需求 |
| **内存** | 容量(GB)、频率(MHz)、类型(DDR4/DDR5) | 多任务处理对内存容量的要求 |
| **硬盘** | 容量(TB/GB)、类型(HDD/SSD)、转速/读写速度 | 系统盘推荐SSD，数据存储可用HDD |
| **显示器** | 分辨率、尺寸、刷新率、色域 | 设计工作需要高色域，游戏需要高刷新率 |
| **网络设备** | 带宽(Mbps/Gbps)、协议标准(802.11ac/ax) | 有线vs无线的场景选择 |

**环节二：操作系统精讲（15分钟）**

1. **操作系统功能框架**
   ```
   操作系统
   ├── 处理器管理（进程调度）
   ├── 存储管理（内存分配）
   ├── 设备管理（驱动程序）
   ├── 文件管理（磁盘组织）
   └── 用户接口（GUI/CLI）
   ```

2. **主流操作系统对比**

| 系统类型 | 代表产品 | 特点 | 适用场景 |
|----------|----------|------|----------|
| 桌面系统 | Windows 10/11 | 生态丰富、兼容性强 | 日常办公、游戏 |
| 桌面系统 | Deepin/银河麒麟/UOS | 国产、安全性高 | 政务、教育国产化 |
| 移动系统 | Android/iOS | 触控交互、应用丰富 | 智能手机、平板 |
| 服务器 | Linux/Windows Server | 稳定、安全 | 企业服务器 |

3. **国产操作系统发展意义**
   - 技术自主创新
   - 信息安全保障
   - 产业链自主可控

**环节三：设备选配案例分析（10分钟）**

案例：某设计工作室需要配置10台电脑，预算50万元，主要用于平面设计（Photoshop/Illustrator）和视频剪辑（Premiere）。

分组讨论：
- CPU如何选择？（Intel i7/i9 vs AMD Ryzen 7/9）
- 内存最低配置？（16GB vs 32GB）
- 显卡是否需要独立显卡？
- 显示器如何选配？

教师点评：
- 平面设计：侧重CPU单核性能、大内存、高色域显示器
- 视频剪辑：侧重CPU多核性能、高速SSD、独立显卡加速

**环节四：操作演示（5分钟）**

虚拟机演示：
- 国产操作系统（Deepin/银河麒麟）界面展示
- 与Windows系统的操作对比
- 应用商店软件安装演示

**环节五：课堂测试（5分钟）**

快速问答：
1. 某CPU标注"3.5GHz/8核16线程"，"8核"指的是什么？
2. 为什么操作系统建议安装在SSD而不是HDD上？
3. 列举2款国产操作系统名称。

### 第3课时：上机综合实践

#### 【实践目标】
1. 通过虚拟机完成操作系统安装体验
2. 完成数制转换与编码计算的上机验证
3. 进行设备配置方案的在线调研与制作

#### 【实践环境】
- 机房计算机（已安装VMware/VirtualBox虚拟机软件）
- 提供国产操作系统镜像文件（Deepin/UOS）
- 提供Windows 10安装镜像（用于对比学习）
- 网络连接（用于在线查询与提交作业）

#### 【实践任务】

**任务一：数制转换验证（20分钟）**

使用计算器工具验证：
1. 打开Windows计算器（程序员模式）
2. 将十进制数2024分别转换为二进制、八进制、十六进制
3. 验证手动计算结果：`(2024)₁₀ = (?)₂ = (?)₁₆`
4. 完成编码计算：计算一张1920×1080分辨率、32位真彩色图像的理论存储容量

提交：数制转换验证截图 + 计算过程文档

**任务二：操作系统安装体验（40分钟）**

分组进行（2人一组）：
- A组：在虚拟机中安装Windows 10，完成基本设置
- B组：在虚拟机中安装Deepin国产操作系统，完成基本设置

操作步骤：
1. 新建虚拟机（分配内存4GB、硬盘60GB）
2. 加载操作系统镜像
3. 完成安装向导（分区、用户名设置等）
4. 安装完成后进行基本设置（分辨率、网络连接）
5. 应用商店中搜索并安装1款办公软件

注意事项：
- 记录安装过程中遇到的问题及解决方法
- 对比两种系统的安装界面差异
- 体验国产操作系统的界面特点

提交：安装完成截图（显示系统版本信息）+ 操作体验心得（200字）

**任务三：硬件配置方案制作（20分钟）**

情境任务：
> 某高三学生需要配置一台5000元左右的电脑，主要用于：
> 1. 日常上网课、做PPT
> 2. Python编程学习
> 3. 偶尔轻度游戏（LOL等）

要求：
1. 使用在线模拟装机平台（如中关村在线DIY装机）
2. 选择合理的CPU、主板、内存、硬盘、显卡、显示器
3. 总价控制在5000±300元
4. 填写配置单并说明选择理由

提交：配置清单截图 + 选配理由说明

#### 【实践评价】

| 评价项目 | 评价标准 | 分值 |
|----------|----------|------|
| 任务完成度 | 三项任务均按要求完成并提交 | 30分 |
| 操作规范性 | 虚拟机使用规范、系统安装步骤正确 | 25分 |
| 方案合理性 | 硬件配置方案性价比高、理由充分 | 25分 |
| 体验反思 | 对国产操作系统的使用有具体感受描述 | 20分 |

## 六、板书设计

```
信息技术基础综合复习

【第一课时板书】
一、信息技术发展
   语言→文字→印刷→电信→计算机→互联网→新一代（AI/5G/IoT）
   
二、信息系统组成
   ┌─────────────────────────┐
   │  硬件 │ 软件 │ 数据 │ 用户  │
   │  ↓    │  ↓   │  ↓   │  ↓   │
   │输入/处理/存储/输出│系统/应用│资源│操作者│
   └─────────────────────────┘

三、数制转换（口诀：除R取余，乘R取整）
   十进制 → 二进制：除2取余（倒序排列）
   二进制 → 十六进制：4位一组，补零分组

【第二课时板书】
四、核心硬件性能指标
   CPU：主频(GHz) × 核心数
   内存：容量(GB) + 频率(MHz)
   硬盘：HDD(容量大/便宜) vs SSD(速度快/贵)

五、操作系统功能框架
   处理器管理 + 存储管理 + 设备管理 + 文件管理 + 用户接口

六、国产操作系统
   代表：Deepin（深度）、银河麒麟、统信UOS
   意义：自主可控、信息安全、国产替代
```

## 七、教学反思建议（教师用）

### 课后反思要点
1. **学生掌握情况**：数制转换计算准确率是否达到80%以上？
2. **上机实践效果**：虚拟机安装过程中学生遇到的主要问题是什么？
3. **时间分配**：三课时内容是否充裕？哪些环节需要调整？
4. **思政融入**：学生对国产操作系统的认同感是否有提升？

### 常见学生问题预判
| 问题 | 应对策略 |
|------|----------|
| 数制转换计算错误 | 强调"除基取余、乘基取整"口诀，增加课堂练习 |
| 虚拟机安装失败 | 提前测试机房环境，准备安装视频备用 |
| 硬件参数不理解 | 准备实物展示（CPU、内存条、硬盘实物） |
| 存储容量计算错误 | 强调"分辨率×色深÷8÷1024÷1024=MB"公式 |
